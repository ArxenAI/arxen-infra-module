# dagon-infra-module — Specification

## Purpose

Reusable OpenTofu modules — the "golden paths" of the platform. Each module is a secure, opinionated building block. Adding a new cloud adds a new provider directory; existing modules are never modified.

---

## Design Principle: Provider-Namespaced Module Directories

```
modules/
  azure/           # MVP — all Azure modules live here
    aks/           # Azure Kubernetes Service cluster
    vnet/          # Virtual Network + subnets
    keyvault/      # Key Vault for secrets
    acr/           # Azure Container Registry
    storage/       # Azure Storage Account (for state, blobs)
    postgresql/    # Azure Database for PostgreSQL Flexible Server
    ai-workspace/  # Azure ML / AI Foundry workspace
    observability/ # Azure Monitor + Log Analytics Workspace
  aws/             # Future — mirrors azure/ structure
    eks/
    vpc/
    secrets-manager/
    ecr/
    s3/
    rds/
    bedrock-workspace/
    observability/
  gcp/             # Future — mirrors azure/ structure
    gke/
    vpc/
    secret-manager/
    artifact-registry/
    gcs/
    cloudsql/
    vertex-workspace/
    observability/
  common/          # Cloud-agnostic modules (no cloud APIs)
    k8s-namespace/ # Kubernetes Namespace + RBAC + NetworkPolicy
    k8s-labels/    # Standard label enforcement
```

## Module Interface Contract

Every module — regardless of cloud — must follow this contract to enable consistent consumption by `dagon-infra-live` and `dagon-templates`.

### Required Files per Module

```
modules/<cloud>/<module-name>/
  main.tf          # Resource definitions
  variables.tf     # All inputs with type + description + default
  outputs.tf       # All outputs with description, sensitive flag
  versions.tf      # required_providers version constraints only (no backend, no provider config)
  README.md        # Generated by terraform-docs
```

### Required Variables (All Modules)

Every module MUST accept these variables:

```hcl
variable "tenant_id" {
  type        = string
  description = "Dagon tenant identifier (internal UUID). Used for tagging and naming."
}

variable "environment" {
  type        = string
  description = "Deployment environment: 'dev', 'stage', or 'prod'."
  validation {
    condition     = contains(["dev", "stage", "prod"], var.environment)
    error_message = "environment must be 'dev', 'stage', or 'prod'."
  }
}

variable "tags" {
  type        = map(string)
  description = "Additional tags to merge with default module tags."
  default     = {}
}
```

### Required Outputs (All Modules)

Every module MUST output at minimum:

```hcl
output "resource_id" {
  description = "The primary resource identifier (ARM ID, ARN, or self-link)."
  value       = <resource>.id
}

output "resource_name" {
  description = "The resource name as provisioned in the cloud."
  value       = <resource>.name
}
```

Modules with sensitive data (connection strings, kubeconfig, tokens) must mark them:

```hcl
output "connection_string" {
  description = "Database connection string. Handle as a secret."
  value       = <resource>.connection_string
  sensitive   = true
}
```

## Azure Module Specifications (MVP)

### `modules/azure/aks/`

**Purpose:** Provision an AKS cluster with secure defaults.

**Key variables:**
| Variable | Type | Description |
|---|---|---|
| `tenant_id` | string | Dagon tenant ID |
| `environment` | string | dev / stage / prod |
| `location` | string | Azure region (e.g., `eastus2`) |
| `subscription_id` | string | Target Azure subscription |
| `kubernetes_version` | string | Kubernetes version (e.g., `1.29`) |
| `node_count` | number | Default node pool size |
| `node_vm_size` | string | VM SKU (e.g., `Standard_D4s_v5`) |
| `vnet_subnet_id` | string | Subnet ID from `azure/vnet` module output |
| `log_analytics_workspace_id` | string | From `azure/observability` module output |
| `tags` | map(string) | Merged with module defaults |

**Security defaults (non-overridable):**
- `private_cluster_enabled = true`
- `azure_policy_enabled = true`
- `oidc_issuer_enabled = true` (for Workload Identity)
- `workload_identity_enabled = true`
- `network_plugin = "azure"` (CNI)
- `network_policy = "calico"`
- Node OS: disk encryption with platform-managed key (CMK in prod)
- RBAC: `role_based_access_control_enabled = true`, local accounts disabled

**Key outputs:** `cluster_id`, `cluster_name`, `kube_config` (sensitive), `oidc_issuer_url`, `kubelet_identity_object_id`

---

### `modules/azure/vnet/`

**Purpose:** Virtual Network with standard subnet layout.

**Subnets provisioned:**
- `snet-aks-nodes` — AKS node pool
- `snet-aks-pods` — AKS pod CIDR (CNI overlay)
- `snet-private-endpoints` — Private endpoints for PaaS services
- `snet-appgw` — Application Gateway (optional)

**Security defaults:** All subnets have NSGs. No public IP on subnets. Private DNS zones linked.

---

### `modules/azure/keyvault/`

**Purpose:** Azure Key Vault for tenant secrets and CMK keys.

**Security defaults:**
- `public_network_access_enabled = false`
- `soft_delete_retention_days = 90`
- `purge_protection_enabled = true`
- Private endpoint required (linked to `snet-private-endpoints`)
- RBAC authorization model (not access policies)

---

### `modules/azure/postgresql/`

**Purpose:** PostgreSQL Flexible Server for `dagon-api` and tenant workloads.

**Security defaults:**
- `delegated_subnet_id` required — no public access
- `private_dns_zone_id` required
- `backup_retention_days = 35`
- `geo_redundant_backup_enabled = true` (stage/prod)
- `ssl_enforcement_enabled = true`
- Encryption with CMK (stage/prod)

## Future Cloud Module Parity (AWS / GCP)

When adding AWS or GCP, mirror the module catalog:

| Azure Module | AWS Equivalent | GCP Equivalent |
|---|---|---|
| `azure/aks` | `aws/eks` | `gcp/gke` |
| `azure/vnet` | `aws/vpc` | `gcp/vpc` |
| `azure/keyvault` | `aws/secrets-manager` | `gcp/secret-manager` |
| `azure/acr` | `aws/ecr` | `gcp/artifact-registry` |
| `azure/postgresql` | `aws/rds` | `gcp/cloudsql` |
| `azure/observability` | `aws/observability` | `gcp/observability` |

**Each new cloud module must implement the same `tenant_id`, `environment`, `tags` variable contract and the `resource_id`, `resource_name` output contract.**

## Versioning

Modules are versioned via Git tags. Callers pin to a tag:

```hcl
source = "git::https://github.com/<org>/dagon-infra-module.git//modules/azure/aks?ref=v2.1.0"
```

**Semver policy:**
- `MAJOR` bump: breaking change to variables or outputs
- `MINOR` bump: new optional variables or outputs
- `PATCH` bump: bug fixes, security patches, doc updates

---

## Non-Goals

- No live environment state in this repo
- No backend configuration blocks
- No provider configuration blocks (only `required_providers`)
- No environment-specific variable values
